## Обоснование исключений
- **Не применимо**: Repudiation для F4 (статистика) - read-only операции не требуют non-repudiation
- **Не применимо**: Elevation of Privilege для F12 - только чтение данных, нет эскалации привилегий
- **Низкий риск**: Tampering для F8 - внутренние вызовы в trusted boundary, минимальный риск изменений

## Ключевые угрозы для Workout Log

### Высокий риск:
1. **Утечка данных тренировок** (F2, F3) - конфиденциальность пользовательских данных
2. **Подмена пользователя** (F1) - доступ к чужим тренировкам
3. **Отказ в обслуживании** (F4) - недоступность статистики

### Средний риск:
1. **Подделка JWT** (F5, F9) - несанкционированный доступ
2. **Отказ от операций** (DB) - проблемы с аудитом
